Docker คืออะไร

Docker เป็น Software Container ที่ถูกพัฒนาขึ้นมาให้สามารถการจัดการ Container ได้ง่าย Image มีขนาดเล็ก แยกเป็นชั้นๆ สร้างแนวคิด build, ship, run ที่แต่ละรอบของการสร้าง Container เร็วขึ้นมาก ทำให้เป็นที่สนใจและกำลังแพร่หลายในกลุ่ม Developer และ System Admin

Docker มีชุดซอฟต์แวร์ให้ใช้งานดังนี้

1. Docker Engine เป็น Core หลักในการทำงาน
2. Docker Machine เป็นซอฟต์แวร์สำหรับสร้าง หรือ เซ็ตอัพเครื่องให้พร้อมสำหรับ container
3. Docker Swarm เป็นซอฟต์แวร์ที่เอาไว้เชื่อมต่อ Docker Engine หลายๆเครื่องมาเชื่อมต่อรวมกันให้เป็น cluster
4. Docker Compose เป็นซอฟต์แวร์สำหรับอำนวยความสะดวกในการรัน Container โดยสามารถสั่งรันหลายๆ Container ได้พร้อม ๆกัน ตั้งค่าต่าง ๆ รวมไปถึงเชื่อมโยง Container ให้สามารถทำงานร่วมกันได้

ทำไมจึงควรใช้ Docker

1. กว่าจะติดตั้ง Software ในเครื่อง Server เพื่อรัน Applicaion ก็เสียเวลาไปกับการติดตั้งและ Configulation กันเป็นชั่วโมงแล้วหรือบางคนอาจจะเสียเวลาเป็นวันๆ
1. เครื่อง Development กับ Production ไม่เหมือนกัน เครื่อง Development ลงโปรแกรมไว้เวอร์ชันนึง แต่บนเครื่อง Production ลงไว้อีกเวอร์ชันนึง บางทีเขียน Code บนเครื่อง Development แล้วสามารถรันได้ปกติ แต่เมื่อเอา Code ขึ้น Production อาจจะทำให้ Code เกิด error หรือไม่สามารถทำงานได้

      ปัญหาดังกล่าวจะไม่เกิดขึ้นถ้าใช้ Docker เพราะ Docker จะช่วยแพ็ค application เหล่านี้อยู่ในรูปแบบของ Container ซึ่งเราสามารถเอา Container ไปรันในเครื่องไหนก็ได้โดย Application ของเรายังทำงานได้ปกติไม่ผิดเพี้ยนจากเดิม ไม่ว่าจะไปรันในเครื่อง Development หรือบน Production Server

องค์ประกอบต่าง ๆ ของ Docker

Docker container container จะถูกสร้างมาจาก Docker Image ที่เป็นต้นแบบ เกิดเป็น container จะได้ Service หรือ Application ที่สามารถเรียกใช้งานได้ทันที
Docker image คือต้นแบบของ Container ข้างในจะเป็น Linux ที่มีการติดตั้ง Application และ มีการ Configuration เอาไว้แล้ว ซึ่งเกิดมาจากการ build ไฟล์ Dockerfile ขึ้นมาเป็น image
Docker registry เราสามารถสร้าง Docker Image แล้วนำไปเก็บรวบรวมไว้บน server (ลักษณะเดียวกับการเก็บ Source Code ไว้บน Github) โดย Docker registry ณ ปัจจุบันก็มีให้เลือกใช้งานได้หลากหลายโดยมี Docker Hub เป็น Docker registry หลักในการเรียกใช้(pull) Docker Image และนอกจากนี้ยังมีผู้ให้บริการ docker registry เจ้าอื่น ๆด้วย เช่น Gitlab, Quay.io, Google Cloud เป็นต้น

จุดเด่นของ Docker

1. ในกระบวน Software Container ทั้งหมด Docker จะใช้ทรัพยากรในการทำงานของตัวมันเอง น้อยที่สุดแล้ว
2. โลกนี้ไม่ได้มี Software Container เพียงแค่ Docker เท่านั้น ยังมีตัวอื่น ๆอีกหลายตัว แต่ Docker ก็ใช้งานได้ง่ายที่สุดแล้ว
3. Docker สามารถ Config ได้ง่าย สามารถสร้าง Container หนึ่งตัวเพื่อรัน Web Service ได้ และสร้างอีกตัว เพื่อรัน MS SQL (คนละแพลตฟอร์มกัน) ได้ โดยใช้ทรัพยากรร่วมกัน
4. สามารถใช้ Docker สร้าง Service แบบเดียวกันหลายๆตัว แล้วให้แต่ละตัวช่วยกันทำงาน ทำให้สามารถรับผู้เยี่ยมชมได้มากขึ้น

ประโยชน์ของ Docker

1. คือการจำลองสภาพแวดล้อมของเครื่องที่ใช้ในการทดสอบให้เหมือนกับเครื่องที่ใช้งานจริงบน Server เนื่องจากปัญหาส่วนใหญ่ที่ Dev เจอในการทดสอบงาน คือสภาพแวดล้อมที่แตกต่างกันของเครื่องจริงและเครื่องทดสอบ ทำให้ได้ผลการทดสอบแตกต่างกัน
2. คือการขยายระบบ ยกตัวอย่างเช่นตอนสิ้นเดือนที่มักจะมีปริมาณการใช้งาน Server มากกว่ากลางเดือน เราก็สามารถใช้ Docker หลายๆตัวช่วยกันทำงานได้ (มีคนช่วยกันทำงานมากกว่า 1 คน งานก็ต้องเสร็จเร็วกว่า หรือไม่ก็สามารถทำงานได้มากกว่า) โดยที่มีค่าใช้จ่ายเพิ่มไม่มากนักเฉพาะตอนที่มีการใช้งานเพิ่มเท่านั้น แลกกับการลงทุนขยายระบบด้วยการเพิ่ม Hardware เพราะใน 1 เดือน อาจจะมีแค่ช่วงเวลาสั้นๆเท่านั้นที่ต้องการทรัพยากรเพิ่ม

ตัวอย่างปัญหา

ปัญหา

สมมุติว่าต้องเขียนเว็บที่ต้องต่อ database จะต้องทำยังไงบ้าง ? อย่างแรกเลยจะต้องติดตั้งโปรแกรมต่าง ๆ ในเครื่องก่อน เช่น คนเขียน php ก็ต้องติดตั้ง Apache หรือ Java ก็ต้องติดตั้ง JDK, Node, .NET เป็นต้น แล้วแต่ว่าจะเขียนเว็บด้วยอะไร ส่วนตัว database ก็ต้องไปติดตั้งเช่นกัน (ตัวอย่าง SQL, MySQL, MongoDB) ขึ้นอยู่กับว่าจะใช้ database ตัวไหน หรือพูดแบบรวบรัดง่าย ๆ คือ ต้องเตรียม Environment ของเครื่องคอมให้พร้อมที่จะทำงาน ตามรูป

แต่เวลาที่ทำงานกันเป็นทีมจริง ๆ ก็จะพบว่าทุกคนในทีมก็ต้องเตรียม Environment แบบเดียวกัน จะได้ทำงานด้วยกันได้ ซึ่งก็จะพบกับปัญหาว่า บางคนใช้ Windows บางคนใช้ MacOS บางคนใช้ Linux และแม้แต่กระทั่ง Software Version ก็ยังเป็นคนละตัวกันอีกต่างหาก ตามรูป

ข้อควรระวัง บ่อยครั้งที่เราไม่สนใจว่าเครื่องแต่ละคนเป็นยังไงใครอยากใช้อะไรก็ใช้ไป มันจะทำให้เรามองไม่เห็นปัญหา เช่น การเขียนโค้ดแบบนี้มันทำงานได้กับ version นี้เท่านั้น พอไปทำงานที่เครื่องอื่นที่ไม่ได้ใช้ version นั้น ๆ

สมมุติว่าเคลียกับคนในทีมจนได้ environments เหมือนกันทั้งทีมละ สุดท้ายเราก็ต้องเอาเว็บที่เราเขียนเสร็จ ไปทำงานที่เครื่องเซิฟเวอร์ซักตัว ซึ่งคำถามคือ เจ้าเซิฟเวอร์เครื่องนั้นมันถูกตั้ง Environments เหมือนกับของเราหรือเปล่า?

สมมุติอีกว่าตั้งค่าต่าง ๆ ของเซิฟเวอร์ได้ไม่มีปัญหา ซึ่งอยู่มาวันหนึ่งคนเข้าชมเว็บเยอะขึ้นเราก็ต้องขยายเซิฟเวอร์เพื่อรองรับผู้ใช้ปริมาณมาก ๆ แล้วเครื่องเซิฟเวอร์ที่เราขยายเพิ่มขึ้นมา เราจะตั้งค่า Environment พวกนั้นยังไง? พวกมันจะคุยกันยังไง? Network ล่ะ? Infrastructure และส่วนอื่น ๆ

วิธีแก้ปัญหา

ซึ่งหนึ่งในการแก้ปัญหาแบบนี้ที่กำลังได้รับการนิยมอยู่ก็คือการใช้ Docker นั่นเอง เมื่อเราหรือทีมจะเขียนอะไรก็ตาม เราก็จะตกลงกันให้เรียบร้อยว่าจะใช้ Environments เป็นแบบไหน แล้วเราก็จะสร้างสิ่งที่เรียกว่า Docker Image ขึ้นมา

Docker Image คล้าย ๆ กับการที่เราทำ Backup Image ที่มันจะจำสถานะทุกอย่างที่เรากำหนดเอาไว้ เช่น ลงโปรแกรมอะไรไว้ เวอร์ชั่นเท่าไหร่ ของต่าง ๆถูกตั้งค่าเอาไว้ยังไง บลาๆ แล้วใครก็ตามที่เอา Image ไปใช้ก็จะได้สถานะ Environments ทุกอย่างตามที่ Image ถูกตั้งค่าไว้นั่นเอง หลังจากที่ได้ Docker Image ละถัดมา ทุกคนในทีมก็จะใช้ Docker Image ตัวนั้นอีกที โดยเจ้า Docker Image จะจัดการ Environments ทุกอย่างให้เป็นไปตามที่มันถูกระบุไว้ ตามรูป

และแน่นอนที่เซิฟเวอร์ก็สามารถเอาไปใช้ได้ และรวมถึงการทำ Server Scaling ก็เช่นกัน
